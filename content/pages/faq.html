This is a question and answer site for questions related to Buddycloud Ltd. and Buddycloud’s Products and Services.

What is Buddycloud?

The Buddycloud stack helps developers add secure communication into their apps. This in-app communication builds community, which then builds a great product experience for the end user. Buddycloud supports developers with reliable tools and knowledgeable `direct-from-the-developer`support.

How does Buddycloud work?

Buddycloud's messaging code handles app authentication, security, cross-device synchronization, offline message management, and more.

Developers add Buddycloud's SDK to their Mobile or Web-apps. This code then communicates with the Buddycloud server to synchronise messages, status-updates, pictures, videos, +1s, and any other message type you can invent.

The Buddycloud service lives in a datacenter and takes care of storing messages and sending real time updates to mobile and web apps as well as talking to other Buddycloud servers.

What are the supported platforms?

Buddycloud has native support for Android and web/Javascript platforms. The backend code runs on any unix-like system. Buddycloud uses Postgres for persistent data.

What distinguishes Buddycloud from other companies providing messaging software?

Buddycloud is the only in-app messaging system that you can also run on-premesis.

Buddycloud believes in an open and free internet and works closely with causes and the standards community to defend our internet freedoms.

Designed for developers or companies?

Buddycloud's tools and services are aimed at developers, as well as, companies. Developers build their products and can host on Buddycloud's infrastructure. Companies receive commercial support, security updates and custom fixes.

What can I build with Buddycloud?

Some apps that have been built include a console-based text messaging, end-to-end secure messaging, a location sharing platform and a social network.

Is Buddycloud free to use?

Free community support, free open source code, built on free and open protocols, free to download and run your own Buddycloud site. This all means you stay free from being locked-in. And, if you need hosting, a free pricing option will get you up and running.

How can I contribute to the open source project?

The Buddycloud community contribute fixes to the open source code and help design future messaging products.

The best place to start is introducing yourself on buddycloud-dev. We love pull requests: each project gets tagged with "Help Wanted" (for example: buddycloud-server).

Buddycloud is Federated. What does that mean?

A federated service means that your users can share messages and updates with users on any other Buddycloud site. This is similar to how a user on gmail.com can send emails to their friend on Yahoo.com: messages flow seamlessly between sites. And like email, it's nice when you can connect with users on remote sites: your Buddycloud site becomes part of something much-much bigger.

Unique hosting options: Medical and financial organisations (and the growing indie-hosting community) are supported with an on-premesis hosting option. All the Buddycloud code is developed in an open and transparent manner making it much more difficult to hide exploitable backdoors in the code.

What privacy controls does the user have?

Users should always know what information is shared and with whom. We've designed Buddycloud to avoid privacy confusion and developed three ways to share:

A selected few trusted users: a private channel, for example - surprise-party@my-domain.com.

Just people on the local domain: a local channel: for example all-hands@company.com

Open to the world: an open channel: for example my-fans@ladygaga.com

What's a username looking like in Buddycloud?

Just like email: Buddycloud usernames describe the user and the site they are from - for example: juliet@capulet.com (users can follow juliets social feed, message her and get her pictures from the capulet.com server).

Can I customize it?

Adding new system components and by inventing new content types mean you can extend Buddycloud to very specific needs. For example, perhaps your application needs to share "Tasks" with team members. You can define a new content type plugin that will validate new tasks. Buddycloud comes with ready made components that handle things like push notifications, media storage, content search. To add new functionality, you simply add a new component to the mix.

What about scaling?

The founding team have worked on massively scaled systems and understand the challenges of designing systems for rapid growth. User growth is supported by horizontally scaling components. Additionally Buddycloud's API includes a plugin for Fanout's realtime CDN to help you globally distribute your traffic.
