Distributed design

Your Buddycloud application plugs into a network of servers also running the Buddycloud stack. Amount of work to enable this: zero.

Your users connect and share with other sites also running Buddycloud. For example a team collaborates and shares documents with a supplier on a remote Buddycloud site. Buddycloud ensures the confidential information stays behind your firewall.

Scalable

Scale up with ease when your application gets popular.

Rapid user growth means you need to quickly scale the Buddycloud backend as simply as spinning up new instances. Adding Realtime CDNs like Fanout is as simple ad editing a config file.

More than just chat or text

Effortlessly add file sharing and soon WebRTC calling between users. The Buddycloud stack takes care of sharing, data synchronisation between users and image previews.

Your app lets users attach content to their chats. All you need to do is use the upload API and the blob is automatically shared with the right people.

No heavy lifting required (Buddycloud does the heavy lifting for your)

Developer quicker by using abstractions for common tasks. Buddycloud has tried to make common tasks as easy as possible to integrate.

Your mobile app should start quickly so Buddycloud helps you by giving you just one sync call that automatically works out what new information needs to be pushed to the mobile client.

Mobile friendly (say no to poll dancing)

Now you can develop messaging that respects your users' mobile battery and data plan. When you integrate buddycloud you benefit from a battery-friendly push notification service and pre-processing and sorting of messages on the server so that our user's batteries last longer and you can do less integration work.

Your app can configure the type of push notifications that should wake it up. Never poll for new messages again.

Open Source

Enjoy the freedom to extend the open source code when you need to. ALL code is released under the liberal Apache 2.0 licence.

Open protocols

Building on open protocols lets you chose from competing Buddycloud server and client implementations. Buddycloud works closely with the W3C, Mozilla Foundation, and XMPP Standards Foundation to build the strong foundation for an open communication ecosystem.

Your project needs a new type of message: Buddycloud uses ActivityStrea.ms for post types and and the ATOM syndication format so you can simply extend the code and use existing libraries.

Secure by design

Build your app on the strong foundation of auditable client libraries, strong client to server crypto and verifiable server code.

Secure communication needs open, inspectable code. You benefit from a community that cares about getting secuirty right (Buddycloud is used by security sensitive agencies). No backdoors, just strong security by design.

Run it yourself or have Buddycloud host it for you.

Buddycloud comes in two compatible flavours: fully hosted and an install-it-yourself set of packages and instructions. Both editions use exactly the same code so that you have the flexibility to migrate your data and users between platforms by just changing the API url.
